[공통 진행 규칙]
- 이번 응답에서는 요청한 단계만 수행해줘.
- 아직 요청하지 않은 다음 단계는 미리 수행하지 말아줘.
- 이전 단계 내용을 불필요하게 길게 반복하지 말고, 필요한 내용만 간단히 참조해줘.
- 이번 프로젝트에서 이전에 확정된 단계별 기록 전체를 기준 문서로 사용해줘.
- 동일 주제의 내용이 여러 단계에 존재하면 가장 최근에 확정된 단계의 내용을 우선 적용해줘.
- 최신 확정본과 이전 단계 내용이 충돌하면 내용을 임의로 섞지 말고 먼저 "충돌 항목"으로 분리해서 정리해줘.
- 누락된 정보는 임의로 확정하지 말고 반드시 "가정" 또는 "추가 확인 필요"로 표시해줘.
- 모호한 표현 대신 개발자가 바로 작업 티켓으로 쪼갤 수 있을 정도로 구체적으로 작성해줘.
- 각 단계 결과는 다음 단계에서 바로 사용할 수 있도록 구조화해서 정리해줘.
- 기능 우선순위는 항상 P0 / P1 / P2 기준으로 유지해줘.
- 사용자 기능과 관리자 기능은 필요한 경우 반드시 구분해줘.
- 실제 서비스 출시 기준으로 보안, 운영, 예외 상황, 권한 구분을 반영해줘.
- 구현 단계(10a, 10b, 10c)에서는 구현 시작 전에 관련 단계 문서 간 충돌 여부를 먼저 점검해줘.
- 구현 단계에서 충돌 항목이 있으면 구현을 시작하지 말고, 충돌 항목과 구현 영향 범위를 먼저 출력해줘.
- 충돌이 해소되기 전에는 API, DB, 인증, 보안, 상태값을 임의로 확정하지 말아줘.
- 구현 단계로 넘어가기 전에 선행 단계가 확정되지 않았다면 구현을 시작하지 말고, 먼저 누락된 확정 항목을 요약해줘.
- 환경변수로 분리해야 할 값은 하드코딩하지 말아줘.
- 단계 마지막에는 반드시 아래 4개를 정리해줘.
    1. 이번 단계 확정 사항
    2. 다음 단계 입력으로 넘길 핵심 요약
    3. 추가 확인 필요 항목
    4. 충돌 가능 항목
