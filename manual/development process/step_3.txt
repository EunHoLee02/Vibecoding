# 3단계 화면 설계 + 사용자 플로우 최종 확정본

# 0. 공통 설계 기준

## 0-1. 화면 설계 원칙

* 웹 우선으로 설계한다.
* 데스크톱 우선 구조를 기준으로 하되, 태블릿/모바일 웹에서도 사용 가능해야 한다.
* P0 화면을 먼저 확정하고, P1/P2는 확장 가능 구조로 둔다.
* 분석 흐름은 반드시 **분석 전 확인 단계**를 거친다.
* 동일 조건 분석 결과가 존재하면 **새 분석 대신 기존 결과 재사용** 흐름으로 분기한다.
* 정식 관리자 UI는 없다.
* 운영 기능은 내부 운영 절차 / 내부 API / 로그 기반 대응으로 처리한다.

## 0-2. 반응형 기준

* **데스크톱**

  * 리스트 + 상세, 입력 + 미리보기의 2컬럼 가능
* **태블릿**

  * 2컬럼 유지 가능하나 CTA 우선 배치
* **모바일 웹**

  * 단일 컬럼
  * CTA 하단 고정 우선
  * 긴 입력 폼은 단계형 또는 블록형으로 축소

## 0-3. 이번 단계 추가 확정 반영

* 회원가입 후 **자동 로그인 하지 않음**
* OCR 중간 처리 상태는 **독립 화면으로 둠**
* OCR 검수 화면에 **임시 저장 기능 포함**
* 분석 실패 시 **재시도 UX 포함**
* 재사용 결과 배지 문구: **`기존 결과 재사용`**
* 재사용 결과 안내 문구:
  **`이전과 동일한 조건의 분석 결과가 있어 새로 분석하지 않고 기존 결과를 보여드려요.`**

# 3. 공통 상태 UX 기준

## 3-1. 빈 상태

* 등록 영양제 없음: 등록 CTA 우선 노출
* 분석 기록 없음: 첫 분석 유도
* OCR 결과 없음: 직접 입력 fallback 제시
* 일부 결과 없음: 표시 가능한 결과 우선 제공

## 3-2. 로딩 상태

* 업로드: 진행률 표시
* OCR 처리: 독립 상태 화면
* 분석 preview: 검증 중 표시
* 분석 실행: 단계형 진행 상태 표시
* 리스트/상세: skeleton 사용

## 3-3. 오류 상태

* 입력 오류: 필드 단위 인라인 메시지
* 권한 오류: 접근 불가 + 홈 이동 유도
* 리소스 없음: 목록 복귀 CTA
* OCR 실패: 직접 입력 유도
* 분석 실패: 재시도 UX + 문의 접수 유도

## 3-4. 성공 상태

* 회원가입 성공
* 로그인 성공
* 영양제 등록 성공
* OCR 임시 저장 성공
* 분석 완료
* 기존 결과 재사용 안내
* 문의 접수 완료

## 이번 단계 확정 사항

1. 3단계 사용자 화면 구조를 확정함
2. 회원가입 후 자동 로그인 없음으로 반영함
3. OCR 중간 처리 상태를 독립 화면으로 확정함
4. OCR 검수 화면에 임시 저장 기능을 반영함
5. 분석 실패 시 재시도 UX를 반영함
6. 재사용 결과 배지/안내 문구를 확정함

   * 배지: `기존 결과 재사용`
   * 문구: `이전과 동일한 조건의 분석 결과가 있어 새로 분석하지 않고 기존 결과를 보여드려요.`

## 다음 단계 입력으로 넘길 핵심 요약

1. 주요 사용자 화면

   * 비로그인 홈
   * 회원가입
   * 로그인
   * 내 영양제 홈
   * 이미지 업로드
   * OCR 처리 중 상태
   * OCR 검수/등록 확정
   * 직접 입력 등록
   * 영양제 상세/수정/보관
   * 분석 전 확인
   * 분석 진행/재사용 분기
   * 분석 결과 상세
   * 분석 기록 목록
