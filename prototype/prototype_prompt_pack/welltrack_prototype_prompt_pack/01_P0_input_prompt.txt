[이번 단계 추가 입력]
- 서비스명: Well Track Prototype
- 한 줄 소개: 사용자가 영양제 정보를 입력하거나 업로드하면 중복 성분, 과다 복용 가능성, 복용 시간 가이드를 빠르게 보여주는 프로토타입
- 프로토타입 목표:
  1. 서비스 핵심 가치가 한눈에 보이게 한다.
  2. 사용자가 3분 이내에 입력 -> 분석 -> 결과 확인 흐름을 체험할 수 있어야 한다.
  3. 로그인 없이 바로 기능을 체험할 수 있어야 한다.
- 주요 사용자:
  1. 여러 영양제를 함께 복용하는 일반 사용자
  2. 중복 성분이나 과다 복용 가능성을 빠르게 확인하고 싶은 사용자
- 핵심 사용자 문제:
  1. 여러 영양제의 성분이 겹치는지 직접 계산하기 어렵다.
  2. 총 복용량 기준으로 과다 가능성을 한눈에 파악하기 어렵다.
  3. 언제 먹는 게 좋은지 정리해서 보기 어렵다.
- 프로토타입에서 반드시 보여줄 핵심 기능:
  1. 영양제 직접 입력
  2. 영양제 이미지 업로드 또는 샘플 업로드
  3. 입력된 영양제 조합 분석 실행
  4. 결과 화면에서 중복 성분, 과다 가능성, 복용 시간 가이드 표시
- 프로토타입에서 허용되는 단순화:
  1. 회원가입/로그인 없음
  2. 관리자 기능 없음
  3. 결제 없음
  4. 문의/운영 기능 없음
  5. 분석은 rule-based 우선
  6. OCR은 실제 연동 대신 stub 또는 mock 허용
  7. 결과 저장은 없어도 됨
  8. 사용자별 데이터 누적 저장은 제외 가능
- 프론트엔드 필요 여부: 예
- 백엔드 필요 여부: 예
- 파일 업로드 필요 여부: 예
- 외부 API 필요 여부: 선택
- 외부 API 정책:
  1. 필수 아님
  2. OCR은 실제 연동보다 mock/stub 우선
  3. 생성형 AI 설명은 제외 가능
- 데이터 저장 방식 후보:
  1. sqlite
  2. 간단한 json/mock data
- 프로토타입 우선순위:
  - P0:
    1. 영양제 입력
    2. 업로드 또는 샘플 입력
    3. 분석 실행
    4. 결과 표시
  - P1:
    1. OCR 정확도 향상
    2. 결과 저장
    3. 분석 설명 강화
  - P2:
    1. 개인화
    2. 관리자
    3. 운영 기능
- 제외할 기능:
  1. 로그인
  2. 회원가입
  3. 비밀번호 재설정
  4. 관리자 페이지
  5. 결제/구독
  6. 문의/오류 신고
  7. 사용자별 기록 누적
  8. 운영 로그
  9. 배포 자동화
- 반드시 유지할 표현 제한:
  1. 의료 진단/처방처럼 보이는 표현 금지
  2. 감기 치료, 질환 개선 보장, 의학적 확정 표현 금지
  3. 일반 가이드 수준으로만 표현
- 프로토타입 핵심 시연 흐름:
  1. 사용자가 영양제 정보를 입력하거나 샘플 이미지를 업로드한다.
  2. 분석 실행 버튼을 누른다.
  3. 결과 화면에서 중복 성분, 과다 가능성, 복용 시간 가이드를 확인한다.
- 결과 화면에서 반드시 보여줄 항목:
  1. 입력된 영양제 목록
  2. 중복 성분 목록
  3. 과다 가능성 표시
  4. 복용 시간 가이드
  5. 요약 한 줄
- 구현 제약:
  1. 빠르게 시연 가능한 구조여야 한다.
  2. 과도한 인증/권한/운영 설계를 넣지 않는다.
  3. endpoint, 필드명, 상태값은 이후 단계에서 바꾸지 않도록 처음부터 단순하고 일관되게 잡는다.
  4. 프로토타입 범위를 넘는 기능 확장은 금지한다.
