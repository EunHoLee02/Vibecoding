[출력 고정 규칙]
- 충돌 또는 미확정 때문에 구현하지 못하면 `파일 단위 구현`, `API별 테스트 케이스 목록`, `이번 턴 생성 파일 목록`, `다음 턴 예정 파일 목록`에 `구현 보류` 또는 `없음`을 명시할 것
- 파일 경로는 오름차순으로 정렬할 것
- API 목록은 method + path 기준 오름차순으로 정렬할 것
- 테스트 케이스는 각 API마다 성공, 권한 실패, 입력 검증 실패, 상태 전이 실패, 외부 연동 실패 순으로 적을 것

    - key1
    - key2
  do_not_carry_forward:
    - key3
    - key4
