아래 조건으로 로그인 없는 웹서비스 프로토타입을 설계/구현하기 위한 프롬프트를 작성해줘.

조건:
- 정식 서비스가 아니라 프로토타입이다.
- 로그인/회원가입/관리자/결제 기능은 제외한다.
- 서비스의 핵심 기능만 프론트엔드와 백엔드로 시연 가능해야 한다.
- 어떤 AI를 사용해도 결과가 크게 흔들리지 않도록,
  화면명, endpoint, 상태값, 주요 필드명을 일관되게 고정해줘.
- 범위를 과하게 키우지 말고 1~3개의 핵심 사용자 시나리오 안에서 끝내줘.
- mock, in-memory, sqlite 중 하나를 명시적으로 선택하게 해줘.
- 기술 스택은 닫힌 비교 방식으로 선택하게 해줘.
- 비어 있는 섹션은 생략하지 말고 반드시 `없음`으로 적게 해줘.
- 마지막에는 반드시 아래를 정리해줘.
  1. 이번 단계 확정 사항
  2. 다음 단계 입력으로 넘길 핵심 요약
  3. 추가 확인 필요 항목
  4. 충돌 가능 항목
  5. 바로 다음 단계 진행 가능 항목
  6. 단계 상태
  7. authoritative_updates YAML

원하는 출력:
- 프로토타입 범위 정의
- 핵심 시나리오
- 화면 구조
- 데이터 모델
- API 계약
- 기술 스택
- 백엔드 구현 프롬프트
- 프론트엔드 구현 프롬프트
- 시연 체크리스트

출력 고정 규칙:
- 위 9개 섹션 순서를 그대로 유지해줘.
- API 계약의 endpoint는 method + path 기준 오름차순으로 정리해줘.
- 기술 스택 섹션에는 후보 비교, raw score, weighted score, 최종 선택 이유를 함께 적게 해줘.
- authoritative_updates YAML은 반드시 ```yaml fenced block```으로 감싸게 해줘.
